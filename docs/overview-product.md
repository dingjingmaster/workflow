# AI Agent 工作流产品文档

> 文档版本：v1.0.0
> 最后更新：2026-05-13
> 更新来源：001-task-workflow-optimization

## 1. 产品定位

本项目提供一套面向 AI Agent 的工程协作工作流，使 Agent 在不同规模任务中按需加载规则、控制上下文成本，并保持需求、计划、实现、验证和总结可追溯。

## 2. 核心价值

- 降低上下文占用：通过入口索引和阶段分片按需加载规则。
- 降低小任务摩擦：L0/L1/L2 使用轻量流程，避免模板填充成本大于任务本身。
- 提高工程可靠性：通过分级、验证、审视和回退机制约束 Agent 行为。
- 保留复用上下文：L2+ 任务进入 `docs/dev/README.md` 索引，后续优先读取 Summary 或轻量任务记录。

## 3. 目标用户

| 用户类型 | 核心需求 | 使用频率 |
|---------|---------|---------|
| 独立开发者 | 让 Agent 少乱改、能验证、能沉淀上下文 | 高频 |
| 小型工程团队 | 用轻量规范统一 AI 协作过程 | 中高频 |
| 复杂项目维护者 | 对跨模块和架构变更保留完整决策记录 | 中频 |

## 4. 工作流边界

- 包含：任务分级、阶段加载、轻量/完整模板、验证约定、回退规则、索引维护。
- 不包含：具体业务系统设计、CI 平台实现、自动化文档生成器。
