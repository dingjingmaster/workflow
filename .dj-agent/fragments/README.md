# 详细设计片段索引

> fragments 只在任务或总文档需要展开细节时使用。默认不加载、不填写。

| 场景 | 片段 | 用途 |
|------|------|------|
| API / 协议 / ABI | `api-design.md` | 记录接口、字段、错误语义和兼容性 |
| 数据 / 配置 | `data-design.md` | 记录数据对象、配置、迁移和生命周期 |
| 发布 / 安装 / 回滚 | `deploy-release.md` | 记录产物、升级、回滚和观测 |
| 性能 | `performance.md` | 记录性能目标、关键路径和验证 |
| 并发 / 锁 | `concurrency.md` | 记录并发模型、锁顺序和竞态验证 |
| 内核 / 驱动 | `kernel-module.md` | 记录内核边界、生命周期、ABI 和验证 |

使用规则：
- 只复制与当前任务直接相关的片段。
- 片段内容应放到 `docs/dev/[N]-*.md` 或长期总文档的相关章节中。
- 与当前任务无关的片段不要填，避免模板噪声。
