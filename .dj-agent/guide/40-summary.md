# 40-summary.md

> 导航：上一节 `30-code.md` ｜ 下一节 `50-review-checklist.md`
>
> 总结阶段：沉淀结论、经验、技术债，更新索引并按需更新总文档。

## 1. 输入与目标

- 输入：当前需求的 Research、Plan、Code 结果与验证记录。
- 目标：形成结构化总结，完成索引更新，便于后续需求按需加载。

## 2. 执行动作

- 记录最终方案与关键决策（含取舍理由）。
- 记录实现结果与计划偏差（若有）。
- 记录问题与处理过程、未解决风险、技术债与后续建议。
- 记录安全门禁执行结果：破坏性操作、工作区保护、高风险项、风险矩阵、命令权限级别、用户确认事项。
- Bug 修复需记录证据等级、复现/回归结果和残余风险。
- L1 可用轻量 Summary 或在最终回复中说明。
- L2 将 Summary 写入 `docs/dev/[ID]-task-[简述].md`。
- L3/L4 产出 Summary 文档（参考 `.dj-agent/003-summary-template.md`）。
- L2+ 更新 `docs/dev/README.md` 索引。
- 按需更新总文档并添加关联链接：
  - `docs/overview-product.md`
  - `docs/overview-product-dev.md`

## 3. 完成标志

- Summary 信息完整且可复盘。
- 多角色审视（按级别裁剪）通过。
- L2+ 索引更新完成，链接可追溯。
- 总文档在确有长期产品或架构影响时已更新。

## 4. 约束

- 总结聚焦“可复用信息”，避免流水账。
- 对不适用章节标注“不适用”，不要为填模板而填模板。
