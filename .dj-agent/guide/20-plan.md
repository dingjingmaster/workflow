# 20-plan.md

> 导航：上一节 `10-research.md` ｜ 下一节 `30-code.md`
>
> 本阶段定义“怎么做”：功能点拆解、技术选型、模块边界、任务清单与验证项。

## 1. 输入与目标

- 输入：Research 结论、相关产品/开发文档、现有代码结构。
- 目标：形成可直接执行的任务计划与验证标准，并更新开发文档。

## 2. 执行动作

- 分析变更范围，明确基于现有功能要修改的模块。
- 做技术选型：优先少依赖、低成本、可维护、可扩展。
- 划分模块职责与依赖关系，确保边界清晰、职责单一。
- 将任务拆成可验证步骤，每步都附带验证项。
- 更新开发文档（默认 `docs/overview-product-dev.md`，无则参考 `.dj-agent/overview-product-dev-template.md` 创建）。

## 3. 完成标志

- 任务清单清晰，执行顺序合理，验证项可落地。
- 开发文档更新完成，且与产品文档一致。
- 三角色审视（按级别裁剪）通过。

## 4. 计划模板建议

```text
1. [步骤] -> 验证：[检查项]
2. [步骤] -> 验证：[检查项]
3. [步骤] -> 验证：[检查项]
```

## 5. 约束

- 计划只包含当前需求所需内容，不扩展未来需求。
- 发现技术不可行或遗漏关键模块时，按附录回退机制处理。
