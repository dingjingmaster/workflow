# 50-review-checklist.md

> 导航：上一节 `40-summary.md` ｜ 下一节 `60-validation.md`
>
> 每个阶段完成后进行三角色审视；不同需求级别按规则裁剪检查项。

## 1. 检查清单

| 角色 | 检查项 | 适用级别 |
|------|--------|---------|
| 高级产品 | ① 是否真正解决了用户问题？ | L2+ |
| 高级产品 | ② 是否存在过度设计或功能蔓延？ | L2+ |
| 高级产品 | ③ 与现有功能是否冲突或重复？ | L2+ |
| 高级产品 | ④ 边界场景和异常流程是否覆盖？ | L3+ |
| 高级产品 | ⑤ 用户体验是否一致？ | L3+ |
| 高级架构师 | ① 方案是否可扩展？是否为未来留有余地？ | L2+ |
| 高级架构师 | ② 是否引入了不必要的依赖或复杂度？ | L2+ |
| 高级架构师 | ③ 模块边界是否清晰？职责是否单一？ | L2+ |
| 高级架构师 | ④ 是否有单点故障或性能瓶颈风险？ | L3+ |
| 高级架构师 | ⑤ 与现有架构的兼容性如何？ | L3+ |
| 高级工程师 | ① 实现是否简洁？有无更简单的方案？ | L1+ |
| 高级工程师 | ② 是否有性能隐患（N+1、内存泄漏等）？ | L1+ |
| 高级工程师 | ③ 错误处理是否充分且不过度？ | L2+ |
| 高级工程师 | ④ 测试是否可验证成功标准？ | L2+ |
| 高级工程师 | ⑤ 代码是否符合项目现有风格和规范？ | L1+ |

## 2. 按级别裁剪

- L0：不加载本清单，只执行最小验证。
- L1：仅做高级工程师检查项 ①②⑤。
- L2：高级产品 ①②③ + 高级架构师 ①②③ + 高级工程师全部。
- L3/L4：全部检查项。

## 3. 审视动作

- 审视发现问题 -> 立即修正 -> 重新审视对应角色 -> 通过后继续。
