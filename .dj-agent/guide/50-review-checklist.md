# 50-review-checklist.md

> 导航：上一节 `40-summary.md` ｜ 下一节 `60-validation.md`
>
> 按级别规定的阶段完成后进行多角色审视；不同需求级别按规则裁剪检查项。安全门禁见 `05-safety-gates.md`。

## 1. 检查清单

| 角色 | 检查项 | 适用级别 |
|------|--------|---------|
| 安全审查员 | ① 是否执行安全门禁，未绕过破坏性操作禁令？ | L0+ |
| 安全审查员 | ② 是否只修改需求范围内文件，且未覆盖用户已有修改？ | L0+ |
| 安全审查员 | ③ Bug 修复是否有复现/证据；E2/E3 是否有回归验证，E1 是否记录相关验证和残余风险？ | L1+ |
| 安全审查员 | ④ 是否存在内存越界、UAF、泄漏、空指针、数据竞争、死锁或资源泄漏风险？ | L2+ |
| 安全审查员 | ⑤ 错误路径是否释放资源并保持状态一致？ | L2+ |
| 安全审查员 | ⑥ ABI/API/协议/配置兼容性是否被破坏？ | L2+ |
| 安全审查员 | ⑦ 回滚或止损方案是否清楚？ | L3+ |
| 高级产品 | ① 是否真正解决了用户问题？ | L2+ |
| 高级产品 | ② 是否存在过度设计或功能蔓延？ | L2+ |
| 高级产品 | ③ 与现有功能是否冲突或重复？ | L2+ |
| 高级产品 | ④ 边界场景和异常流程是否覆盖？ | L3+ |
| 高级产品 | ⑤ 用户体验是否一致？ | L3+ |
| 高级架构师 | ① 方案是否满足已知扩展需求，且未引入不必要复杂度？ | L2+ |
| 高级架构师 | ② 是否引入了不必要的依赖或复杂度？ | L2+ |
| 高级架构师 | ③ 模块边界是否清晰？职责是否单一？ | L2+ |
| 高级架构师 | ④ 是否有单点故障或性能瓶颈风险？ | L3+ |
| 高级架构师 | ⑤ 与现有架构的兼容性如何？ | L3+ |
| 高级工程师 | ① 实现是否简洁？有无更简单的方案？ | L1+ |
| 高级工程师 | ② 是否有明显性能或资源隐患？ | L1+ |
| 高级工程师 | ③ 错误处理是否充分且不过度？ | L2+ |
| 高级工程师 | ④ 测试是否可验证成功标准？ | L2+ |
| 高级工程师 | ⑤ 代码是否符合项目现有风格和规范？ | L1+ |

证据式审查要求：
- L2+ 或命中高风险门禁时，关键检查项不能只写“通过”，必须记录证据、结论和未覆盖风险。
- 执行证据式审查时，对安全审查员 ③④⑤⑥，必须引用复现结果、代码路径、测试命令、静态检查或人工审查结论之一。

## 2. 按级别裁剪

- L0：不加载本清单，但必须执行 `05-safety-gates.md` 和最小验证。
- L1：安全审查员 ①②③ + 高级工程师检查项 ①②⑤。
- L2：安全审查员 ①②③④⑤⑥ + 高级产品 ①②③ + 高级架构师 ①②③ + 高级工程师全部。
- L3/L4：全部检查项。

## 3. 审视动作

- 审视发现问题 -> 按风险决定修正或回退；涉及范围扩大、C2/C3、破坏性操作、用户已有修改冲突或需求不清时，先回退到对应阶段或获取用户确认 -> 重新审视对应角色 -> 通过后继续。
