# 代码提交说明模板

> 用于提交代码时填写 commit message / 提交注释。内容必须具体、可验证；副作用或风险没有时写“无”。

```text
[类型] 修复 xxx 问题 / 实现 xxx 功能

1. 修复/实现
- 修复 xxx 问题 / 实现 xxx 功能：
- 关联需求/问题：

2. 代码逻辑改动
- 改动的模块/文件：
- 改动的核心逻辑：
- 新增/删除/调整的行为：

3. 影响的使用场景
- 影响场景：
- 不影响场景：

4. 副作用/风险
- 副作用：
- 风险：
- 风险应对/验证：
```

最小示例：

```text
[fix] 修复网关重复启动时状态判断错误

1. 修复/实现
- 修复网关重复启动时误判为未运行的问题。
- 关联需求/问题：用户反馈重复执行 start 后状态不一致。

2. 代码逻辑改动
- 改动的模块/文件：internal/gateway/runner.go
- 改动的核心逻辑：启动前先读取已有 pid 并校验进程状态。
- 新增/删除/调整的行为：重复启动时直接返回当前运行状态。

3. 影响的使用场景
- 影响场景：重复执行 gateway start。
- 不影响场景：首次启动、停止、状态查询。

4. 副作用/风险
- 副作用：无。
- 风险：无。
- 风险应对/验证：go test ./internal/gateway
```
